How YouTube Downloaders Work (Technically)
The Full Architecture
User pastes YouTube URL
        ↓
Tool sends request to YouTube (mimicking a browser)
        ↓
YouTube returns a page with encrypted stream URLs
        ↓
Tool decrypts the "signature cipher" (YouTube's obfuscation)
        ↓
Tool fetches the actual video/audio stream directly
        ↓
Merges video + audio using ffmpeg → saves file
The 3 Core Technical Problems & How They're Solved
Problem 1 — Signature Cipher (encryption)

YouTube encrypts stream URLs with a JavaScript function that changes every few days. Downloaders solve this by:

Downloading YouTube's player JS file
Parsing and executing the decryption function
Applying it to decode the real stream URL

Problem 2 — Bot Detection

YouTube detects bots by checking:
SignalHow Bots Get CaughtNo browser cookiesReal users always have cookiesMissing HTTP headersBrowsers send 20+ headers automaticallyRequest rateBots request too fastJavaScript not executedBots skip JS, browsers run itNo TLS fingerprint matchPython's TLS handshake differs from Chrome's
Solutions (Research/Educational):
Method 1 — Send real browser headers
  User-Agent, Accept-Language, Sec-Fetch headers etc.

Method 2 — Use a real browser engine
  Playwright / Puppeteer — runs actual Chrome/Firefox
  YouTube cannot tell the difference

Method 3 — Rotate requests
  Don't hammer the same URL repeatedly

Method 4 — PO Token (YouTube's newest defense)
  YouTube now requires a "Proof of Origin" token
  Generated by running their JavaScript in a real browser
  yt-dlp solves this via browser cookies OR
  via a local token generator
Problem 3 — Format Selection

YouTube streams video and audio separately (DASH format). Tools must:

List all available formats
Let user pick quality
Download video stream + audio stream separately
Merge with ffmpeg

How Snaptube-Style Apps Work
Snaptube is a mobile app that essentially bundles a full browser engine + a download interceptor:
Architecture:
┌─────────────────────────────┐
│   WebView (real browser)    │  ← User browses normally
├─────────────────────────────┤
│   Network Interceptor       │  ← Catches video stream URLs
├─────────────────────────────┤
│   Format Picker UI          │  ← Shows quality options
├─────────────────────────────┤
│   Download Manager          │  ← Saves with resume support
└─────────────────────────────┘
The key insight: they don't bypass anything — they just catch the stream URL that the browser was already going to load, then download it directly instead of just playing it.
